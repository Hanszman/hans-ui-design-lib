# Hans UI Color System

This page documents the color tokens from `src/styles/colors.scss`.

## Structure

Each color family is defined as a trio:

- `900`: strong
- `500`: default
- `300`: neutral

OBS: The Color Gray has five values: 900, 700, 500, 300, 100

These trios are mapped to component variants via semantic tokens:

- `--primary-*`
- `--secondary-*`
- `--success-*`
- `--danger-*`
- `--warning-*`
- `--info-*`
- `--base-*`

## Fallback Theme

`:root` defines fallback semantic values.  
If no `data-theme` is set, components use this mapping.

## Theme Combinations

Themes are activated with:

```html
<body data-theme="combination3"></body>
```

Available combinations:

- `combination1`
- `combination2`
- `combination3`
- `combination4`
- `combination5`

Each combination remaps semantic tokens (`primary`, `secondary`, etc.) to different base trios.
And we can also define the color for background and text in our combinations:

- `--background-color`
- `--text-color`

## Usage Note

Components should always consume semantic tokens (`--primary-default-color`, `--base-neutral-color`, etc.) instead of raw family tokens (`--purple-500`, etc.).  
This keeps themes swappable without changing component SCSS.
