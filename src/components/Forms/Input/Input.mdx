The **Input** component is used to pass information in forms of the interface.
It supports multiple sizes and colors variations, allowing full customization with themes.

## Usage

To use the component you need to import it and include it in your React app.
Make sure you have already set up the [Hans UI Design Library](/docs/getting-started--docs) in your project.

```tsx
import { HansInput } from 'hans-ui-design-lib';

export const Example = () => (
  <HansInput label="Input" inputColor="base" inputSize="medium" />
);
```

## Web Component Events

When used as `<hans-input>`, the component exposes normalized value events from
`onValueChange`. Prefer `valuechange` in Angular templates because it is
case-insensitive friendly:

```html
<hans-input leftIcon="LuSearch" (valuechange)="search($event.detail)"></hans-input>
```

For compatibility, the Web Component also dispatches `valueChange` and
`value-change`. The event detail is always the current input value, so framework
consumers can update local state without parsing React or native event objects.

For trailing icon actions, prefer `righticonclick` in Angular templates:

```html
<hans-input
  rightIcon="LuEye"
  rightIconAriaLabel="Show password"
  (righticonclick)="togglePasswordVisibility()"
></hans-input>
```

For compatibility, the component also dispatches `rightIconClick` and
`right-icon-click`.
